ICE & Stage

Background to ICE

ICE was established by the music societies PRS, STIM and GEMA to facilitate collaboration between music societies, music publishers and digital service providers (e.g. Spotify, Apple Music, Youtube etc).

Project Overview

I was involved in the AV Project, which focuses on the use of music in audiovisual productions - such as films and TV series. Our team worked to create a “Master Data Management” system for collecting, storing and sharing audiovisual data. The high-level purpose of this system is to ensure that composers and songwriters are accurately paid for the use of their works in AV productions.

My Involvement

I worked both remotely and on-site in Berlin for ICE, for 20hrs/per week, whilst simultaneously completing a degree in History at the University of York.
In April 2019, the project was outsourced from ICE to Stage Enterprises. I then worked for Stage entirely remotely until April 2020, when I ceased working on the project in order to complete my final exams.

UI & UX Process

1. Identify Common User Actions or “User Scenarios” and Create Storyboards

Example: “a user wishes to match a production with the correct match from a list of possible candidates”

After conducting interviews with users and reading through the system requirements documentation,* we identified simple “user scenarios” that detailed key actions a user might perform. From these “user scenarios,” I created basic wireframes (above), in order to determine how, when and where certain information should be displayed.

*Although we worked as an Agile Scrum team, the project was initially set up following a “waterfall” approach.

2. Create Prototypes & Present to Customers

Using these very basic wireframes and insights gained from user interviews, I created prototypes of screens using Sketch. In the process, we identified design questions and challenges, which we documented, proposed potential solutions for, and presented to our customers.

–EXAMPLE DOCUMENTATION

Scenario

"A user wishes to match a production with the correct match from a list of possible candidates.” (example continued)
Basic Approach

Our basic idea for the match conflict user interface is to show the most relevant information of the conflicting production on the left-hand side and a list of the candidates on the right-hand side.

Additionally, there needs to be one button for linking the production to the candidate, as well as one button for creating a new production instead.

However, this solution raises several questions and challenges...

Challenges & Proposed Solutions

Problem: with a long list of candidates, the last one might be placed with some distance from the actual production, making it might be hard to compare the values.

Problem: when the user needs to compare more data fields, the simple view becomes insufficient. How can more details be displayed on demand?

Solution: whenever the user selects a candidate the original production is moved and aligned next to in. In order to maintain the object tracking of the user, this is done with an animated transition (see below).

Solution: there is a “more” button that collapses further production details on demand (see below).

I created this animation using the AnimateMate plugin for Sketch.

3. Develop the Components & Get Feedback from Users

We then developed the components deployed them for testing, taking feedback from users. We identified bugs that needed to be fixed, as well as enhancements that should be carried out.

Reflections

My experience at ICE and Stage was incredibly rewarding. From working in an entirely new city (Berlin), to tackling difficult design decisions and having to quickly learn new technologies, I was constantly pushed to the edge of my comfort zone and learnt a vast amount in the process.

Key Learnings

  • Data Visualisation - Schneiderman's mantra: "Overview first, zoom and filter, then details on demand." We used this to guide our design decisions.

  • Tools and Technologies - I frequently used Sketch. I was also introduced to the highly useful Adobe Colour Picker tool and the Bootstrap CSS Framework.

  • Responsive Design - Amongst my "user stories" was to ensure every page we created was responsive, using flex grids and layouts and media queries.

  • Time-Management - as I spent 20hrs/week on the project whilst completing my (full-time) History degree, I greatly developed my ability to effectively manage my time and to work efficiently. I turn, I had loads of time leftover to spend having fun with my friends and learning new things.
